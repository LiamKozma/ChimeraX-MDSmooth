Command: mdsmooth

Usage

mdsmooth  structure  [toAtoms atom-spec] [alignAtoms atom-spec] [reference frame] [signal rmsd|pc1|ic1|dpca|deeptica] [lag N] [seeds N] [mode extrema|kinetic] [states N] [microstates N] [stateSteps N] [ticaDim N] [targetFrames N] [cutoffFrequency freq] [powerCutoff fraction] [order N] [includeEnds true|false] [makeMorph true|false] [method corkscrew|linear] [savePdbs directory] [showPlot true|false] [name string] [extraFrames frame,frame,…]

The mdsmooth command reduces a molecular dynamics trajectory to the frames that carry its structurally meaningful motion, then builds a morph of just those frames. By default it computes the per-frame RMSD to a reference frame, applies a zero-phase low-pass Butterworth filter to remove high-frequency noise, finds the local maxima and minima of the filtered signal (the “significant frames”), and interpolates between them with per-segment step counts chosen so the morph keeps the trajectory's original timing. Interpolation runs through ChimeraX's own morph engine (corkscrew) by default, so rotating domains follow a screw motion rather than sliding in straight lines.

Arguments

structure (required)
A single atomic model containing a trajectory (multiple coordinate sets), e.g. #1.
toAtoms atom-spec
Atoms whose signal is filtered to select the significant frames. Default: every atom of structure except bulk solvent and free ions (water, Na+, Cl−, and so on), which diffuse freely and would otherwise swamp the macromolecule's own motion. Protein, nucleic acid, ligands, and nonstandard residues are all kept by default (see Ligands and nonstandard residues below).

Pass an explicit spec to focus the analysis on a region. This is how you track, say, a binding pocket rather than the whole protein:

mdsmooth #1 toAtoms #1@CA                    # backbone C-alphas only
mdsmooth #1 toAtoms #1:45,88,120,145         # specific pocket residues
mdsmooth #1 toAtoms #1 & #1:LIG :< 5        # everything within 5 Å of ligand LIG
alignAtoms atom-spec
Optional. By default the morph is not superimposed on anything. It keeps the raw simulation coordinates, so it stays registered on the input trajectory and drifts and tumbles exactly as the raw does, and the smoothing only removes the high-frequency jitter. This is what lets you overlay the morph on the original trajectory and watch them together. Pass alignAtoms only if you deliberately want to remove global motion by superimposing those atoms onto the reference frame (e.g. alignAtoms #1:1-435@CA to de-tumble on the protein). Note that superimposing on a selection holds it still, so aligning on the very atoms you want to watch move would pin them.
reference frame
1-based frame number used as the RMSD reference. Default 1 (first frame), matching cpptraj's rms first.
signal rmsd|pc1|ic1|dpca|deeptica
Which 1D signal the significant frames are picked from.

rmsd (default) uses each frame's RMSD to the reference. It is the robust choice: it always computes, has no tuning knobs, and does not depend on how well the trajectory samples any one motion.

pc1 uses the projection onto the first principal component of the toAtoms selection, the single largest collective motion (essential dynamics), computed after rigid-body superposition so it captures internal motion rather than tumbling.

ic1 uses the projection onto the first time-lagged independent component (tICA), which is the slowest collective motion rather than the largest. Slowness is often a better proxy for a functional transition than variance, since large thermal breathing has high variance but no function, so IC1 can recover a change that PC1 buries under fast wobble. Its lag is set by lag, and IC1's shape is checked for stability across lag times (an unstable slow mode is cautioned as possibly undersampled).

dpca uses dihedral PCA, the first principal component of the backbone φ/ψ angles, sin/cos encoded to handle their periodicity. Because dihedrals are internal coordinates, dPCA needs no structural alignment and is immune to global tumbling, a good choice for flexible or looping backbones where Cartesian PCA is dominated by large lever-arm swings. It requires protein backbone atoms.

deeptica is a learned nonlinear slow CV: a DeepTICA or state-free reversible VAMPnet trained with PyTorch and mlcolvar, which can capture slow motions a linear tICA misses. It is an optional, heavy feature. PyTorch runs in a dedicated environment that you install once with mdsmooth installLearnedCV, and if it is missing the command tells you so. Because learned CVs are initialization-unstable, DeepTICA trains several seeds (see seeds) and takes their consensus, and cautions you if the seeds disagree. At modest data scales the linear ic1 signal is usually the safer choice; prefer DeepTICA only when you have plenty of sampling with real transitions.

PC1, IC1, dPC1, and DeepTICA track the direction of the motion rather than mere distance from a reference, and cost more to compute, so keep toAtoms focused. Everything downstream (filtering, frame selection, morph) is identical for all signals.

All signals report a cosine content (0 to 1). A value in the diffusive regime (above 0.85) means the signal resembles a random walk, so on a single short trajectory the “slow mode” may reflect undersampled diffusion rather than a sampled transition, and the log and preview flag it as a caution. It is a diagnostic, not a verdict: a genuine slow sweep can look cosine-like too.

lag N
Lag time in frames for the ic1 (tICA) signal and for kinetic mode; default 10. This is tICA's one finicky knob: too small behaves like PC1 (fast modes leak in), too large gets noisy (few lagged frame pairs remain). Must be at least 1 and smaller than the number of frames. Ignored for the rmsd and pc1 signals.
seeds N
For the deeptica signal only: how many independently-initialized models to train and take the consensus of; default 5. More seeds give a steadier consensus (and a better disagreement estimate) but take proportionally longer.
mdsmooth installLearnedCV [gpu true|false] [indexUrl url]
A separate one-time command that installs the optional learned-CV environment (PyTorch and mlcolvar) into a dedicated virtualenv, not ChimeraX's own Python, following the pattern of ChimeraX's built-in ML tools. This downloads several GB and takes a few minutes. gpu true selects a CUDA torch build; indexUrl overrides the pip index (e.g. a specific https://download.pytorch.org/whl/cuXXX or …/whl/cpu). Only needed for the deeptica signal.
mode extrema|kinetic
How key frames are chosen. extrema (default) is the signal pipeline above: filter a 1D signal and cut at its local maxima and minima. kinetic is a different strategy. It clusters the trajectory into kinetically distinct metastable states (tICA, then k-means microstates, then a Markov State Model, then PCCA+ coarse-graining) and picks one representative frame per state, ordered along the slowest coordinate into a tour of the distinct conformations. It answers “show me the N different shapes” rather than “sample the main motion”. Kinetic mode ignores the cutoff controls (targetFrames, cutoffFrequency, powerCutoff) and the signal keyword. It needs the optional deeptime package; if it is missing the command says so and how to install it (pip install deeptime).
states N
Kinetic mode only: how many metastable states (representative key frames) to extract; default 5. If PCCA+ finds fewer populated states than requested, you get as many as it found.
microstates N
Kinetic mode only: how many k-means microstates the Markov State Model is built on before coarse-graining; default 100. Clamped to the range [states, number of frames − 1].
stateSteps N
Kinetic mode only: interpolation frames per morph segment; default 20. Because the representatives are ordered by conformation rather than time, each state-to-state transition simply gets equal screen time.
ticaDim N
Kinetic mode only: how many slow tICA components the clustering runs in; default 4.
targetFrames N
Ask for about N significant frames and let the command solve for the low-pass cutoff that produces them. This is the recommended dial: it is the most intuitive way to control the movie's detail, it doubles as a speed control (more frames means more morph segments to build, so fewer frames build faster on a slow machine), and it adapts to the toAtoms selection. It re-fits the cutoff to whatever signal you hand it, so the same targetFrames gives a comparable movie whatever atoms you track. Default: 50 when no cutoff is given at all. The count is not capped, so ask for as many as you like. Because the number of frames is a stepwise function of the cutoff, the result lands close to N rather than exactly on it; the log reports the number actually found.

The three controls below all set the same underlying low-pass cutoff. If you give more than one, they are honoured in priority order (targetFrames, then cutoffFrequency, then powerCutoff), and the command logs a warning saying which one it used.

cutoffFrequency freq
Set the low-pass cutoff explicitly, in cycles per frame (must be between 0 and 0.5). Lower values smooth harder and yield fewer significant frames, which is useful for reproducing a specific published movie. Overrides powerCutoff; overridden by targetFrames.
powerCutoff fraction
Choose the cutoff as the frequency that retains this fraction of cumulative spectral power (mean removed first). Has no default, so supply it only if you want spectral-power-based selection instead of a frame target. Overridden by targetFrames or cutoffFrequency.
order N
Butterworth filter order. Default 5.
includeEnds true|false
Include the first and last frames as significant frames even if they are not extrema, so the morph spans the whole trajectory. Default true.
makeMorph true|false
Build a new morph trajectory from the significant frames. Default true. Set false to only report the frames.
method corkscrew|linear
How to interpolate between consecutive significant frames. Default corkscrew: each segment is interpolated through ChimeraX's morph engine, which decomposes the change into a rigid-body screw motion plus internal-coordinate change, so rotating domains turn along an arc instead of atoms sliding through one another. linear is a straight-line interpolation in Cartesian space. It is faster and has no ChimeraX dependency, but it can stretch bonds and flatten rings across large conformational jumps (see Carbohydrates and other ring systems). Either way the per-segment frame counts (and hence the movie's timing) are the same; only the path between key frames differs. If the morph engine is unavailable, corkscrew automatically falls back to linear.

Because corkscrew runs the morph engine once per segment, it takes longer to build on larger structures and at a higher cutoffFrequency (a higher cutoff keeps more significant frames, and hence more segments to morph). Linear is nearly instant either way.

savePdbs directory
Also write one PDB per significant frame into directory (created if needed), named <name>_frame0001.pdb and so on using the original 1-based frame numbers. These are the exact key frames the morph interpolates between, handy for inspection or for morphing by hand. Off by default.
showPlot true|false
Pop up a graph of the raw and filtered signal with the significant frames marked (maxima red, minima blue, and any user-added frames as green diamonds on the unfiltered trace). Default true. Ignored when ChimeraX is run without a graphical interface (--nogui). The plot is interactive; see Adding your own significant frames.
name string
Name for the created morph model. Default smoothed.
extraFrames frame,frame,…
Extra significant frames to force into the morph in addition to the ones the filter finds, given as a comma-separated list of 1-based frame numbers (the same numbering as reference), e.g. extraFrames 250,600. Use this when you know a conformation matters but the filtered signal is flat there so no extremum is detected. A frame that is already significant is left as-is (no duplicate). Frames added this way are labelled User in the results table and drawn as green diamonds on the plot. You can also add them by clicking; see below.

Adding your own significant frames

Sometimes a conformation you care about sits on a flat stretch of the filtered signal, so it is not picked up as a maximum or minimum. You can add it yourself in two ways:

  • By command: pass extraFrames with the 1-based frame numbers, e.g. mdsmooth #1 cutoffFrequency 0.01 extraFrames 250,600.
  • By clicking the plot: in the plot window, click Add significant frame, then click on the graph where you want the frame. A green preview marker lands on the unfiltered (grey) trace so you can see exactly which frame you are about to add; click again to move it, then click Add frame N to keep it. The morph is rebuilt in place to include the new key frame, and the overlay and playback link follow the rebuilt copy.

Ligands and nonstandard residues

Selection is by atom, so ligands and nonstandard residues are treated exactly like anything else. In the default toAtoms set they are kept (only bulk water and free ions are removed), so a bound ligand's motion contributes to the signal and appears in the morph. To analyze a ligand on its own, name it: toAtoms #1:LIG. To watch a pocket including its ligand, use a zone: toAtoms #1 & #1:LIG :< 5. The whole structure, ligands and ions included, is still carried through the morph regardless of toAtoms, which only chooses which atoms' signal picks the significant frames.

Carbohydrates and other ring systems

Sugar rings and other small rings have preferred shapes (for a pyranose, the chair) that matter for their function, so it is worth knowing where MDSmooth can and cannot affect them.

The frames MDSmooth selects are never altered. The significant frames, and the PDBs written by savePdbs, are exact snapshots from your trajectory, so their ring shapes are whatever the simulation produced. The only synthetic geometry is the in-between frames of the morph, and a ring can look distorted there only when its pucker changes between two neighbouring key frames. Corkscrew, the default, interpolates local geometry rather than dragging atoms along straight lines, so it preserves ring shape far better than linear, which can flatten a ring as it flips. For carbohydrate-heavy systems, keep the default corkscrew method and read the in-between frames as illustration; the real geometry lives in the key frames.

None of the signals tracks ring puckering directly (dPCA uses protein backbone dihedrals, not sugar rings), so if a pucker transition is itself the motion you want to show, pick the frames around it by hand with extraFrames.

Viewing and playing the results

Each build adds the smoothed morph as a new model (#2, #3, and so on) alongside the raw trajectory (#1), with a descriptive name such as smoothed (IC1, 50 key frames). Use the Model Panel or info models to see which number is which.

  • Play one morph: open its own Coordinate Sets slider (coordset slider #2) and press play. The plot opened by the command scrubs the raw model, not the morph.
  • Play the raw and a smoothed copy together, in sync: coordset #1,2 1,1901 loop true (use your trajectory's frame count). This works because an extrema-mode morph preserves the raw's timeline, so they stay aligned frame for frame. It is a good way to compare RMSD against IC1 against PC1 side by side: coordset #1,2,3 1,1901. Stop with coordset stop.
  • Kinetic tours are the exception: a mode kinetic morph is a short conformation sequence (about states×stateSteps frames), not a timeline, so play it on its own slider (coordset slider #2) rather than syncing it to the raw.
  • Grey-ghost overlay (raw faded under the smoothed copy): easiest from the MDSmooth panel, which also links playback automatically. By hand: transparency #1 60 target acbps ; color #1 gray target ac.
  • Record a movie of any of the above with the movie command.

Examples

open trajectory.pdb
mdsmooth #1
mdsmooth #1 toAtoms #1@CA cutoffFrequency 0.01 name clean
mdsmooth #1 toAtoms #1 & #1:LIG :< 5 method corkscrew   # binding-pocket motion, corkscrew morph
mdsmooth #1 signal ic1 toAtoms :436-441 targetFrames 50    # slowest binding-site motion (tICA)
mdsmooth #1 mode kinetic states 5 toAtoms :436-441         # tour of 5 metastable states (needs deeptime)
mdsmooth #1 method linear savePdbs ~/Desktop/keyframes    # linear morph plus export the key frames
mdsmooth #1 cutoffFrequency 0.01 extraFrames 250,600      # force two extra frames into the morph
coordset #1,2 1,1901 loop true                              # play raw and smoothed together, in sync

The command logs a table of significant frames (frame number, filtered signal value, type, and interpolation steps) and adds the morph as a new model, which can then be played and recorded with ChimeraX's movie command.
